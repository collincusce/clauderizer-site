# Clauderizer × Kimi Code CLI setup

Generated by `clauderize init`; rewritten on every run; safe to delete. Kimi Code
CLI is the successor to the legacy Kimi CLI (which used `~/.kimi/`); it serves the
Kimi K3 model and uses `.kimi-code/` for config.

## 1. MCP server — already wired (auto-written)

`clauderize init` registers the `clauderizer` server in this repo's
`.kimi-code/mcp.json` (project-level; the same `mcpServers` shape Cursor uses),
non-destructively. Nothing to do — the cz_* tools are available on next launch.
The portable command it registers is:

```
uvx --from clauderizer[mcp] clauderizer-mcp
```

## 2. Orientation — automatic via the MCP server

`clauderize init` writes the Clauderizer stanza into `AGENTS.md` (a
`<!-- clauderizer -->` marker block). If your Kimi Code CLI reads `AGENTS.md`, the
"call `cz_status` first" floor is loaded from there. **Either way**, the Clauderizer
MCP server attaches a one-line status note to the **first tool call** of a session
(server bootstrap), so the agent is oriented even without `AGENTS.md` or hooks —
this is the reliable automatic path on Kimi Code CLI. Wiring the hooks below just
makes it arrive at session start instead.

## 3. Session-start hooks — add to `~/.kimi-code/config.toml`

Kimi Code CLI adds a hook's stdout to context on exit 0. These four events carry
the Clauderizer status digest (one command serves all four; the engine routes on
`hook_event_name` and stays silent when it has nothing to add). Clauderizer does
NOT edit your global `~/.kimi-code/config.toml` — merge this yourself:

```toml
[[hooks]]
event = "SessionStart"
command = 'uvx --from clauderizer clauderizer-hook'

[[hooks]]
event = "UserPromptSubmit"
command = 'uvx --from clauderizer clauderizer-hook'

[[hooks]]
event = "PreCompact"
command = 'uvx --from clauderizer clauderizer-hook'

[[hooks]]
event = "PostCompact"
command = 'uvx --from clauderizer clauderizer-hook'
```

## 4. Skills — expose them to Kimi Code CLI

Kimi Code CLI loads Agent Skills from `.kimi-code/skills/`, `.agents/skills/`, and
`~/.agents/skills/` — NOT from `.claude/skills/`, where `clauderize init` installs
Clauderizer's skills. To use them in Kimi Code CLI, expose that directory, e.g.:

```
mkdir -p .agents/skills && ln -s ../../.claude/skills/* .agents/skills/
```

(or copy them). This step is optional — the MCP tools and the AGENTS.md floor work
without it.
